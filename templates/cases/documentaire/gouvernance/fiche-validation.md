# Fiche de validation

Un harnais sans fiche de validation signée reste un **prototype**, quel que
soit son affichage.

| Élément | Valeur |
|---|---|
| Statut demandé | <prototype \| interne \| production> |
| Périmètre | <Périmètre couvert par cette validation> |
| Relu par (fonction) | <Fonction ayant relu, jamais un nom> |
| Date de relecture | <AAAA-MM-JJ> |
| DPO informé | <Oui / Non — fonction DPO> |
| DSI / RSSI associés | <Oui / Non — fonction DSI/RSSI> |
| Réserves | <Réserves éventuelles> |

<Rappelez ici que le passage à un usage réel suppose des sources réelles
validées par leurs propriétaires et une nouvelle fiche de validation signée.
Cette validation ne vaut pas validation juridique.>
